# Contributing to react-a11y-toolbar

Thanks for your interest in contributing! Before opening a pull request,
please read this document — particularly the section on licensing and the
Contributor License Agreement (CLA), because it affects every contribution.

## TL;DR

- This project is dual-licensed: **AGPL-3.0** for open-source use, and a
  **separate commercial license** for organizations that need it.
- To keep that dual-licensing model workable, all non-trivial contributions
  require a signed **Contributor License Agreement (CLA)**. Our CLA bot will
  prompt you on your first pull request.
- We aim to be transparent about this upfront so there are no surprises.

## Why a CLA?

Dual-licensing only works if a single party has the rights to offer the code
under both licenses. If a contributor merges AGPL-licensed code without
granting the maintainer relicensing rights, that code can't be included in
the commercial offering — which over time fragments the project into "core
(maintainer-owned)" and "community (mixed ownership)," which is bad for
everyone.

Our CLA is a **license grant**, not a copyright assignment. You keep the
copyright to your contribution. You grant the Licensor a broad license to
use, modify, and sublicense your contribution under both AGPL-3.0 and any
other license (including the commercial license). Your contribution also
remains freely available to the community under AGPL-3.0.

We know CLAs are sometimes controversial. If you don't want to sign one,
that's a legitimate choice — but we can't accept the contribution in that
case. We'd rather you know this upfront than discover it at PR-merge time.

## Signing the CLA

- Individual contributors: the CLA bot will post a link on your first PR.
  It's a one-click signature via GitHub and only needs to be done once.
- Contributions made on behalf of an employer: a corporate CLA signed by
  someone authorized to bind the company is required. Please email
  yaron.rosenbaum@gmail.com to arrange this before submitting code.

## Contribution licensing, in plain English

When you contribute code, documentation, translations, tests, or any other
copyrightable material to this project, you agree that:

1. You have the right to submit it (you wrote it, or you have permission
   from the rights-holder).
2. Your contribution is licensed under AGPL-3.0 to the community.
3. The Licensor (Yaron Rosenbaum) may also distribute your contribution
   under the project's commercial license and under any future license the
   Licensor chooses, subject to the CLA you signed.

If a contribution is too small to be copyrightable (typo fixes,
single-character changes, trivial whitespace corrections), we won't block
it on CLA status — though the bot may still ask you to sign before your
next contribution.

## What makes a good pull request

- Open an issue first for anything beyond a small fix, so we can discuss
  the approach before you spend time on code.
- Keep PRs focused: one concern per PR.
- Add or update tests.
- Update documentation when behavior changes.
- Follow the existing code style (linter + formatter configs are in the
  repo).
- For accessibility-related changes, include notes on:
  - which assistive technologies you tested with (NVDA, VoiceOver, JAWS,
    TalkBack, etc.);
  - which WCAG success criteria the change addresses;
  - any known limitations or trade-offs.

## Code of conduct

Be kind. Assume good faith. Disagreements about technical direction are
welcome; personal attacks are not. Maintainers may close or lock
discussions that stop being productive.

## Reporting security issues

Please do **not** file public issues for security vulnerabilities. Email
yaron.rosenbaum@gmail.com with details. See `SECURITY.md` (if present) for the
full process.

## Questions?

Open a discussion on GitHub, or email yaron.rosenbaum@gmail.com. We'd rather answer
licensing questions upfront than have contributors feel blindsided later.
